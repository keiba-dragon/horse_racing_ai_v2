## 前提条件（必ず守ること）

### 1. 新馬除外（全セグメント共通）
- **新馬（デビュー戦）を全セグメントから除外する**
- 理由: 新馬は前走情報が存在しない → 前走系特徴が全てNaN → モデルの歪み
- **全5セグメントとも `クラス_rank != 1.0` で除外済み ✅**
- 新馬除外後は `1走前_馬場状態`, `間隔`, `1走前_3角` 等のNaN率が大幅に下がる

### 2. 使用する特徴量はJVLink APIで確実に取れるもののみ
- 出馬表（前日〜当日12時）から取得できる特徴量に限定
- 調教データはNG（APIで確実に取れない）
- 賞金カラムはリーケージ禁止

### 3. NaN率の管理
- 新馬除外後も残るNaN → **キャリア上の限界**（例: 3走前クラス差がNaNなのはまだ3戦目の馬）は許容
- キャリア上のNANに対しては `_isnan` 指示変数を自動追加してモデルに組み込む
- **新馬除外で消えるNaN（デビュー戦由来）に `_isnan` を付けるのは不適切**

### 4. 特徴量は最低10個
- greedy forward selectionは「全候補がNaN」の場合のみ停止
- 改善なしでも△マークで追加継続（タイ防止）
- FORCED_BASE = ['馬番', '斤量']（常に含める）

### 5. モデル構造
- **clogit（条件付きロジット）+ isotonic calibration のみ**
- lambdarank・サブモデル一切不使用
- L2 = 0.006, Adam optimizer, early stopping on val NLL

### 6. 選択指標
- **2325 = (r2324×n2324 + r25×n25) / (n2324+n25)、単年±30%キャップ付き**
- train: 2013-2021, val: 2022, OOS: 2023-24, 2025, 2026
- 報告値: 2023-24, 2025, 2026, 25+26 combined

### 7. 比較ルール
- モデル変更前に必ず旧モデルを同じparquetで再評価してからベースラインとする
- 比較は同じparquet・同じ学習期間・同じOOS期間で揃える

## 重要な技術メモ

### baba_map（馬場状態の数値変換）
```python
baba_map = {'良': 0, '稍重': 1, '重': 2, '不良': 3}
# load_segment()内で必ずbaba_mapを先に適用してからpd.to_numericを呼ぶ
# 逆順にするとpd.to_numericが文字列を全NaN化してbaba_mapが無効になる
```

### NaN指示変数の追加条件
```python
NAN_IND_THRESHOLD = 0.05
# trainデータでNaN率 > 5% の特徴 f に対して f_isnan = 0/1 を自動追加
# 新馬除外後はNaN率が下がるため、指示変数が必要な特徴が減る
```
