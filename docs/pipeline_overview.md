## 2. モデル: 芝ダ別 条件付きロジット (Surface Conditional Logit)

**コアの確率モデル**:

```
P(馬i が勝つ) = exp(Xi × β) / Σ_{j ∈ 同レース} exp(Xj × β)
```

各レース内でのソフトマックスなので「同レース内での相対強さ」を学習する。

**特徴量**:
- 336列 (数値列から現在レースのオッズ系列を除外)
- lambdarankの重要度上位35特徴量の交互作用項（2次）を追加
- 主要特徴: 1走前クラス調整着順、芝ダ一致平均着順、近3走平均着順 など

**学習**: Adamオプティマイザ + L2正則化、val NLL で early stopping

**事後補正**: Isotonic Regression (val データで fit) → 確率を単調に再マッピング

---

## 3. 予測から投票対象選択まで

### ステップ1: Raw確率
```python
raw_prob = softmax(X @ beta)  # レースごと
```

### ステップ2: キャリブレーション
```python
calib_prob = isotonic.predict(raw_prob)
```

### ステップ3: ランキングスコア (最終モデルの工夫)
```python
is_maiden = (クラス_rank == 2)   # 未勝利かどうか

factor = 0.00  if is_maiden else 0.16

score = calib_prob - factor * (1 / odds)
#              ↑              ↑
#        モデル確率      市場の過剰評価分を差し引く
```

### ステップ4: 選択
```
各レースで score が最大の馬に単勝100円を購入
```

---

## 4. ROIの計算式

```
ROI = Σ(的中時のオッズ × 100) / (全レース数 × 100) - 1
```

- 控除率なしの理論値はROI = 0%
- 実際の単勝控除率は約20% → ランダム選択なら ROI ≈ −20%
- **目標**: −12% (控除率20%から8ポイント改善)

## 6. 最終モデルのポイント (なぜ hybrid が効いたか)

**未勝利 (factor=0)**: 未勝利は市場が非効率なため、モデル確率だけで選ぶのが最良。
オッズで調整すると余計な情報が混入して悪化する。

**非未勝利 (factor=0.16)**: 上位クラスでは市場が比較的効率的。`1/odds`（市場確率）の
16%分を差し引くことで、オッズが過剰に低い人気馬を避ける。

**効果**: 勝率は 25.4% → 24.1% に下がるが、的中時のオッズが上がりROIが改善。

## 10. 主な制約・注意事項

- **リークなし**: 現在レースの単勝オッズは特徴量から除外 (`ODDS_REMOVE`)
- **芝ダ別モデル**: `距離` 列の先頭文字で判別 (`芝` or `ダ`)
- **Isotonic は ranking に無影響**: 単調変換なので rank=1 選択には影響しない。hybrid ranking の scale 調整にのみ意味がある
- **統計的誤差**: 11,559レースでSE ≈ ±1.5%。−12.5% と −12.0% は誤差範囲内
