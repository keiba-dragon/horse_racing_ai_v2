モデルが示す3つの発見
発見 1 / 最大の逆説
「直近に勝った馬」より
「コンスタントに入る馬」が強い
近3走で勝ちがある馬（勝率高い）はむしろ次走で勝ちにくい。一方で3着以内が続く馬は次も勝ちやすい。
近3走_複勝率: β = +0.271（3着以内継続 → 買い）
近3走_勝率: β = −0.176（直近に勝利 → 逆に売り）
ダート長距離では斤量増・クラス昇級の反動が出やすく、「勝った馬の次走」より「惜しい2〜3着が続く馬」の方が信頼できる。
発見 2 / タイム指数
「今の調子」より
「本来の実力水準」が支配的
タイム指数のピーク（max）と平均がほぼ同じ重みで採用。一方でトレンド（加速度）はほぼ無効。
近5走_タイム指数_max: β = +0.375
近5走_タイム指数平均: β = +0.367
タイム指数_加速度: β = −0.035（ほぼゼロ）
「最近上昇中か下降中か」よりも「そもそも速い馬かどうか」が重要。ダート長距離は能力の絶対値がレースを決める。
発見 3 / 最後の一押し
「相手レベル_平均着順」が
30%突破の鍵だった
35特徴（29.80%）→ 36特徴（30.02%）の差は「相手レベル_平均着順」1特徴の追加のみ。
相手レベル_平均着順: β = +0.133
追加後: 29.80% → 30.02%（+0.22pp）
クラス補正着順との違いは「そのレースにいた全馬の水準」を反映する点。強い相手と戦い続けてきた馬を正しく評価する。
発見 4 / 血統
「種牡馬_ダ_勝率」は
「種牡馬_勝率」で代替可能
ダート専用血統特徴量（種牡馬_ダ_勝率）を追加したが β=0.000 で全く無効。
種牡馬_勝率（全体）: β = +0.090（有効）
種牡馬_ダ_勝率: β = +0.000（無効）
「ダート専用血統」は「全体血統」と高相関のため追加情報ゼロ。このセグメント（ダート長距離）では全体血統だけで十分。

  • v1 — 初期探索（10特徴）

    新候補（タイム指数トレンド系・輸送有無・コース脚質等）を追加して全候補から再探索。 greedy selectionは局所最適に依存するため、シードを変えることで異なる解を発見。
    28.86% ↑+0.43pp
  • v4 — スタミナ特徴追加（失敗）
    前走長距離フラグ等を追加したが acc_2325 が低下。
    27.99% ←v3より悪化
  • v5 — v3シードでMAX_FEATS=30（30特徴）
    v3の25特徴を起点にさらに5特徴追加。近10走_勝率・2走前_クラス差・道悪着順等が採用。

  • v6 — 新候補15種追加・MAX_FEATS=35（35特徴）
    未使用だった特徴量（馬_r20_勝率・騎手会場_r100_勝率・性別_num・近3走_勝率等）を発掘・追加。 馬_r20_勝率(β+0.086)・騎手会場_r100_勝率(β+0.111)・近3走_勝率(β−0.176)が採用。
    29.80% ↑+0.72pp（最大の1回改善）
  • v7 — MAX_FEATS=40（36特徴）🎯 目標達成
    最後に 相手レベル_平均着順(β+0.133) が1特徴追加されて30%突破。 探索総時間 118秒。
    30.02% ✅ ↑+0.22pp
